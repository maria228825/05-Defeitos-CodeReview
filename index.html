Bug Report Profissional + Code Review Guiado

Esta página facilita o uso do GitHub Pages no fork da dupla. A atividade combina exploração de uma aplicação com defeitos injetados e revisão estática de código, gerando um relatório final único para entrega.

Duração total: 1h Formato: em duplas Entrega final: PDF no Teams

Parte A — Aplicação para testes

Abra o sistema TarefaQS para explorar cenários, reproduzir defeitos e registrar pelo menos 3 bug reports completos.

Parte B — Código para revisão

Acesse o arquivo JavaScript da revisão estática e use o checklist para registrar no mínimo 6 findings com linha, rótulo e severidade.

Abrir código para revisão Ver roteiro da Parte B

Como usar esta página

  1. Ative o GitHub Pages no fork com a pasta raiz /(root).
  2. Abra a URL principal do seu fork no navegador.
  3. Use os botões acima para acessar a aplicação de testes e o código da revisão.
  4. Preencha os arquivos markdown do repositório e consolide tudo em entregavel/RELATORIO.md.

Objetivo pedagógico

A proposta é treinar duas competências centrais de QA: documentar defeitos com precisão e identificar riscos diretamente no código, antes da execução.

Ao final, exportem o relatório preenchido para PDF e submetam o arquivo na atividade correspondente no Microsoft Teams.

Se algum link acima abrir o Markdown como texto cru, isso é esperado no GitHub Pages. Nesse caso, use o mesmo caminho diretamente no repositório do fork para editar o arquivo.
